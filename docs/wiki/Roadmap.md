## Release Cycle

### Major Versions
- 2-3 major versions per year
- Include significant feature updates
- Require migration guides

### Minor Versions
- 1-2 minor versions per quarter
- Include new features and improvements
- Backward compatible

### Patch Versions
- Released as needed
- Fix bugs and security issues
- Fast iteration

## Version Support Strategy

- **Current Version**: Full support
- **Previous Version**: Security updates and major bug fixes
- **Earlier Versions**: Security updates only
